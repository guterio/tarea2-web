Ocultar

¿Qué te pedimos que hagas?

Para realizar correctamente la tarea hay que hacer las siguientes actividades:

1.- Documento de análisis de requisitos.

  • El documento de especificaciones debe estar basado en los requisitos que el cliente ha indicado (recuerda que si no estás realizando el proyecto para una empresa, entonces el cliente eres tú), indicando todo lo que se quiere conseguir con el proyecto.
  • Muy importante. Este documento será de una página como máximo.

2.- Diseño del diagrama  de casos de uso

  • Obtención de cada uno de los casos de uso y el Diagrama de Casos de Uso
  • Ten en cuenta que la aplicación puede utilizar diferentes roles

3.- Diseño de la interfaz de la aplicación

  • Realización de  diagrama en el que se muestre cada una de las subpáginas y página principal de las que constara la aplicación web, así como los elementos que contienen, conexiones, enlaces y el flujo de datos que existe entre ellas. No se exige herramienta específica para dicho trabajo. (Solo para aplicaciones Web)
  • Diseño de las pantallas y el flujo de una a otra (Solo para aplicaciones de escritorio o Dispositivos Móviles)

4.- Diseño de la base de datos

  • El diseño de la Base de Datos debe incluir el modelo Entidad – Relación, su paso al modelo relacional y los script de creación de tablas (se deberá normalizar si se cree que es necesario).
  • Crea un script para la creación de la base de datos.
  • Crea un script que inserte datos en la BD a partir de un conjunto de datos inicial sobre el que poder ir realizando las primeras pruebas. Se debe apoyar en un modelo de datos en el cuál existan al menos 6 tablas, en las que al menos debe existir 3 claves externas indicando relación de unas con otras. Se deben crear los usuarios, privilegios y roles necesarios.

5.- Diagrama de clases (en el caso de que la aplicación no utilice una base de datos).

  • Se realizará el diseño de las clases más importantes de la aplicación.

6.- Necesidades hardware y software necesarias para la aplicación

  • Elección justificada de la infraestructura hardware / software necesaria para correr la aplicación. Indicar los recursos de interconexión necesarios para montar dicha infraestructura. Indicar las alternativas existentes al diseño elegido e indicar posibles mejoras.

Ocultar

Recuerda que el proyecto, debería contener incondicionalmente aparte de lo que creas necesario:

  • Sistema de autenticación
  • Sistema de gestión (mas de uno que interaccionan) con altas, modificaciones, bajas, listar, búsqueda.
  • Interfaz completa de varias pantallas
  • Varios niveles de acceso
  • Todo lo necesario para cubrir lo indicado en este ERS y posibles situaciones derivadas de los requerimientos

 

Tras la entrega del documento de especificaciones el profesor podrá proponer cambios para ajustarse a la dificultad de un proyecto de estas características.

  • Proyecto DAM: Las aplicación a desarrollar debe ser un software de escritorio o una aplicación para dispositivos móviles Android.
  • Proyecto DAW: La aplicación a desarrollar debe ser una aplicación web.
